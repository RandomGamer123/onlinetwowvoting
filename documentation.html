Modes

  • signup - Outputs contestant information in format [id,last_known_username,timestamp] to the google sheet.

  • respond - Allows contestants to respond, contestants are defined by a json list. The responses will be outputted in format [id,last_known_username,timestamp,response] to the google sheet.

  • signup-r - Allows anyone to respond, automatically signing them to the json list. The responses will be outputted in the same format as respond

  • respond-d - Allows anyone to respond, but not adding them to the json list, intended for dummy responses. The responses will be outputted in the same format as respond

  • vote - Allows anyone to vote, their votes will be outputted in format [id,last_known_username,timestamp,vote] to the google sheet.

  • nothing - Does not allow anyone to do anything.

  • hiatus - Does not allow anyone to do anything. Displays a yellow banner saying that this minitwow is on hiatus on the relevant page.

  • cancelled - Does not allow anyone to do anything. Displays a red banner saying that this minitwow is cancelled on the relevant page.

  • ended - Does not allow anyone to do anything. Displays a green banner saying that this minitwow has ended on the relevant page.

  • x-[custom] - Custom mode, by default does not do anything. [custom] can be replaced by any string for the maximum of 8 characters, contact me if you need custom implementations.

Voting Screen Generation Modes

  • t - Traditional Mode, uses a link to voting screen(s).

  • n - New Mode, a dynamic system of generating screens based on a user inputted screen name.
